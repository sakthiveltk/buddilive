Buddi Live Encryption Details

When using Buddi Live, your privacy and data is protected by a number of factors:

  1. Your user ID (email address) is stored as a cryptographically hashed value. This means that it is not feasible for anyone to discover your email address by looking at the database. Note that if you opt to 'Store Email', your email will be stored in plain text in another field. If you are worried about your privacy, do not click the 'Store Email' option.
  2. If you opt to encrypt the data, then certain fields in the database are encrypted using a symmetric-key cryptographic system (currently implemented as 256-bit AES, although this may change in the future). Fields which are encrypted include Account / Budget Category Name, Account Type, Transaction Description (Payee), Transaction Number (Cheque Number), and Transaction Memo. Note that the amounts are not encrypted (due to technical limitations); however, since the account ID is hashed, and the account / transaction information is encrypted, the amounts by themselves are meaningless.

Note that by enabling encryption, there is no way to reset a lost password. There are no back doors to the system. If you forget your password, your data is not recoverable.
